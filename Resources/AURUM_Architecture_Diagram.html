3
Core Intelligence Layer — The 4 AURUM Engines
AURUM Intelligence Core — All engines run in parallel on each routing event
Engine 01
APEX
Acuity Prediction Engine
Transforms raw vitals + symptoms into a full care pathway map. Outputs: severity score, required equipment, specialist type, and per-hospital Survivability-by-Routing Score.
Input: BP, HR, SpO2, GCS, age, incident type, comorbidities
XGBoost model → care dependency map
SHAP values → explainability output
Survivability % per routing option
XGBoost SHAP scikit-learn ONNX export Platt scaling
Engine 02
ORACLE
Predictive Hospital State Engine
Maintains a 30-minute forward projection of every hospital's capacity. Routes to where care WILL be available — not just where it is now. The Digital Twin of the hospital grid.
Ingest: discharge orders, OR schedules, shift rosters
ARIMA/Prophet time-series projection
State machine per hospital: P(available, t+N)
Feeds NEXUS with future-state grid
statsmodels Prophet Redis TS State machine
Engine 03
NEXUS
Constraint Optimization Router
ILP-based optimizer solving the multi-constraint assignment problem. Single patient or mass casualty batch. Triggers pre-arrival activation at receiving hospital.
Consume APEX care map + ORACLE future state
OR-Tools ILP: minimize risk × transit time
MCE mode: batch N patients → M hospitals
Publish decision → Pre-Arrival Brief
OR-Tools PuLP (ILP) NetworkX OSRM client
Engine 04
HELIX
Adaptive Learning Loop
Closes the feedback loop. Every routing outcome retrains APEX. Federated learning — hospitals share gradient updates, not raw patient data. PDPB/HIPAA compliant by design.
Collect: routing outcome, patient survival, prediction error
Federated: hospitals send gradients, not data
Central aggregator retrains APEX model
Push analytics to admin dashboard
Flower (flwr) Opacus DP PyTorch Diff. Privacy
↓  Engines read/write to persistent data stores   ↓

Critical Data Flows — Single Patient Scenario vs Mass Casualty Event
Flow A — Single Patient Routing
1
EMT speaks vitals → Whisper.cpp parses → JSON sent to FastAPI /predict
2
APEX → XGBoost predicts care pathway + per-hospital Survivability Score
3
ORACLE → fetches future state grid from Redis (30-min projections)
4
NEXUS → OR-Tools ILP solves → optimal hospital decision in <500ms
5
Pre-Arrival Brief sent → Hospital Terminal via Web Push
6
Routing decision + Survivability Score → EMT mobile display + Dispatch Dashboard
7
Post-outcome: HELIX collects result → gradient update → APEX retrains
Flow B — Mass Casualty Event (MCE)
1
MCE declared by dispatcher → NEXUS switches to Batch Optimization Mode
2
APEX runs parallel predictions for all N incoming patients simultaneously
3
ORACLE projects surge capacity impact if N patients hit any single hospital
4
NEXUS ILP: assign N patients → M hospitals. Hard constraint: no hospital > surge cap
5
Ripple recompute: manual override of 1 patient → all others re-optimized (<2s)
6
Grid Allocation View appears on Dispatch Dashboard — split screen, drag-and-drop
7
N Pre-Arrival Briefs dispatched to N hospitals simultaneously via WebSocket
